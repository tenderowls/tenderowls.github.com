Repository

First of all make sure that you added our Maven Repository into setting of your project or maven global settings.xml.

<pluginRepositories>
    <pluginRepository>
        <id>haxemojos-public-repository</id>
        <url>http://repo.haxemojos.org/content/groups/public</url>
    </pluginRepository>
</pluginRepositories>

<repositories>
    <repository>
        <id>haxemojos-public-repository</id>
        <url>http://repo.haxemojos.org/content/groups/public</url>
    </repository>
</repositories>

Plugin hosted in the Central, but Haxe compiler and Neko runtime artifacts not because the Central allows to publish only jar artifacts with java classes.

Project

Add plugin into build section.

<build>
    <sourceDirectory>src/main/haxe</sourceDirectory>
    <plugins>
        <plugin>
            <groupId>com.tenderowls.opensource</groupId>
            <artifactId>haxemojos-maven-plugin</artifactId>
            <version>1.0.0</version>
            <extensions>true</extensions>
            <configuration>
                <swfVersion>10</swfVersion>
                <main>com.example.project.Main</main>
            </configuration>
            <dependencies>
                <dependency>
                    <groupId>org.haxe.compiler</groupId>
                    <artifactId>haxe-compiler</artifactId>
                    <version>3.0.0-rc</version>
                    <type>pom</type>
                </dependency>
                <dependency>
                    <groupId>org.nekovm</groupId>
                    <artifactId>nekovm</artifactId>
                    <version>2.0.0</version>
                    <type>pom</type>
                </dependency>
            </dependencies>
        </plugin>
    </plugins>
</build>

Don’t be afraid haxe-compiler dependency has pom type. This is a meta-dependency: plugin recognize your OS and configure type and classifier automatically.
